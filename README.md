# AI Engineering Workflow

> 用 AI 作为开发主力，独立推进中大型项目的多角色协作工作流。

## 概述

本项目定义了一套**角色分工 + 文件驱动**的 AI 辅助开发工作流，专门面向使用 AI（如 Trae IDE）独立推进中大型项目的开发者。

核心思想：**用文件代替记忆，用分工代替混用。**

## 角色分工

| 角色 | 职责 | 核心文件 |
|------|------|----------|
| **启动者** | 从零到一，通过引导式对话挖透需求，输出初始化文档 | `启动者.md` |
| **架构师** | 决策层，读文件、分析现状、输出执行指令 | `架构师.md`、`架构师启动.md` |
| **操盘手** | 执行层，接收指令、修改代码、反馈结果 | `操盘手.md`、`操盘手启动.md` |

## 工作流程

```
启动者（需求挖掘）
    ↓
架构师（分析现状 → 输出指令）
    ↓
操盘手（执行开发 → 反馈结果）
    ↓
架构师（验证产出 → 下一条指令）
    ↻ 循环直至完成
```

## 核心文件

| 文件 | 说明 |
|------|------|
| `工作流总览.md` | 整体工作流规范、窗口分工、核心文件说明 |
| `启动者.md` | 启动者角色规范与引导式交互原则 |
| `架构师.md` | 架构师角色规范、分析原则、沟通方式 |
| `架构师启动.md` | 架构师首次启动流程与执行指令格式 |
| `操盘手.md` | 操盘手角色规范与执行原则 |
| `操盘手启动.md` | 操盘手首次启动流程与标准操作流程 |

## 使用方式

1. 将对应角色的 `.md` 文件作为 AI 助手的 System Prompt 或 Skill 加载
2. 按角色分工使用对应窗口，不跨角色操作
3. 维护好 `架构.md` 和 `进度.md` 两份核心文件，驱动整个闭环

## 适用场景

- 使用 AI 作为开发主力，独立推进中大型项目
- 代码基础有限、依赖 AI 做技术判断的开发者
- 需要多人（多窗口）协作的复杂开发任务
